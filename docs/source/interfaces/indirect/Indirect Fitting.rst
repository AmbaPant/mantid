<p>Each interface provides the option to choose between selecting one or multiple data files to be fit.
The selected mode can be changed by clicking either the 'Single Input' tab or 'Multiple Input' tab at the the top

Data may either be provided as a file, or selected from workspaces which have already been loaded.</p>

<a name="user-content-sequential-and-simultaneous-fits"></a>
<h2><a id="user-content-sequential-and-simultaneous-fits" class="anchor" aria-hidden="true" href="#sequential-and-simultaneous-fits"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="#id121">Sequential and Simultaneous fits</a></h2>
<p>There are options to fit your selected spectra either Sequentially or Simultaneously.</p>
<p>A <a href="#id1"><span id="user-content-id2">:ref:`sequential fit &lt;algm-IqtFitSequential&gt;`</span></a> will fit each spectra one after another. By default this will use the end values of one fit as the starting values of the next. This behaviour can be toggled
using the sequential/individual option.</p>
<p>A <a href="#id3"><span id="user-content-id4">:ref:`simultaneous fit &lt;algm-IqtFitSimultaneous&gt;`</span></a> will fit all the selected spectra against one cost function. The primary advantage of this method is that
parameters which are expected to be constant across the data range can be tied across all the spectra. This leads to these parameters being
fitted with better statistics and hence reduced errors.</p>

<p>Under 'Fit Function', you can view the selected model and associated parameters as well as make modifications.</p>
<p>There are two modes which can be used to select functions. The default version allows easy selection of the most commonly used function models. The options
in this mode differ for each of the four fitting tabs so more detailed information is given in the specific sections below. The other mode, which may be switched to

<p>Parameters may be tied by right-clicking on a parameter and selecting either 'Tie &gt; To Function' to tie the parameter

<h2><a id="user-content-settings" class="anchor" aria-hidden="true" href="#settings"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="#id123">Settings</a></h2>
<p>The bottom half of the Fit Function section contains a table of settings which control what sort of fit is done. These are:</p>
<dl>
<dt>Fitting</dt>
<dd>Controls whether to do a sequential or simultaneous fit.</dd>
<dt>Max Iterations</dt>
<dd>The maximum number of iterations used to perform the fit of each spectrum.</dd>
<dt>Minimizer</dt>
<dd>The minimizer used for the fit (defaults to Levenberg-Marquadt).</dd>
<dt>Cost function</dt>
<dd>The cost function used for the fit (defaults to Least Squares).</dd>
<dt>Evaluate Function As</dt>
<dd>The way the function is evaluated on histogram data sets. If the value is “CentrePoint” then function is evaluated at centre of each bin. If it is “Histogram” then function is integrated within the bin and the integrals returned.</dd>
<dt>Fit Type</dt>
<dd>Only available in sequential fits. Controls whether each fit is initialised from the end values of the previous fit.</dd>
</dl>

<p>The results of the fit may be plotted and saved under the 'Output' section of the fitting interfaces.</p>
<p>Next to the 'Plot' label, you can select a parameter to plot and then click 'Plot' to plot it with error
bars across the fit spectra (if multiple data-sets have been used, a separate plot will be produced for each data-set).
The 'Plot Output' options will be disabled after a fit if there is only one data point for the parameters.</p>
<p>During a sequential fit, the parameters calculated for one spectrum are used as the initial parameters for the next spectrum to be fitted.
Although this normally yields better parameter values for the later spectra, it can also lead to poorly fitted parameters if the
next spectrum is not 'related' to the previous spectrum. It may be useful to replace this poorly fitted spectrum with the results
from a single fit using the 'Edit Result' option.
Clicking the 'Edit Result' button will allow you to modify the data within your <em>_Results</em> workspace using the results
produced from a fit to a single spectrum. See the algorithm <a href="#id5"><span id="user-content-id6">:ref:`IndirectReplaceFitResult &lt;algm-IndirectReplaceFitResult&gt;`</span></a>.</p>
<p>Clicking the 'Save Result' button will save the result of the fit to your default save location.</p>

<h3><a id="user-content-msd-fit" class="anchor" aria-hidden="true" href="#msd-fit"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="#id127">MSD Fit</a></h3>
<p>Given either a saved NeXus file, or workspace generated using the Elwin tab, this
tab fits <tt>intensity</tt> vs. <tt>Q</tt> with one of three functions for each
run specified to give the Mean Square Displacement (MSD). It then plots the MSD
as function of run number. This is done using the
<a href="#id7"><span id="user-content-id8">:ref:`QENSFitSequential &lt;algm-QENSFitSequential&gt;`</span></a> algorithm.</p>
<p>MSDFit searches for the log files named &lt;runnumber&gt;_sample.txt in your chosen
raw file directory (the name ‘sample’ is for OSIRIS). These log files will exist
if the correct temperature was loaded using SE-log-name in the Elwin tab. If they
exist the temperature is read and the MSD is plotted versus temperature; if they do
not exist the MSD is plotted versus run number (last 3 digits).</p>
<p>The fitted parameters for all runs are in _msd_Table and the &lt;u2&gt; in _msd. To
run the Sequential fit a workspace named &lt;inst&gt;&lt;first-run&gt;_to_&lt;last-run&gt;_eq is
created, consisting of <tt>intensity</tt> v. <tt>Q</tt> for all runs. A contour or 3D plot of
this may be of interest.</p>
<p>A sequential fit is run by clicking the Run button at the bottom of the tab, a
single fit can be performed using the Fit Single Spectrum button underneath the
preview plot. A simultaneous fit may be performed in a very similar fashion by changeing the Fit Type to Simultaneous
and the clicking run.</p>
<p>The <a href="#id9"><span id="user-content-id10">:ref:`Peters model &lt;func-MsdPeters&gt;`</span></a> [1] reduces to a <a href="#id11"><span id="user-content-id12">:ref:`Gaussian &lt;func-MsdGauss&gt;`</span></a> at large
(towards infinity) beta. The <a href="#id13"><span id="user-content-id14">:ref:`Yi Model &lt;func-MsdYi&gt;`</span></a> [2] reduces to a <a href="#id15"><span id="user-content-id16">:ref:`Gaussian &lt;func-MsdGauss&gt;`</span></a> at sigma
equal to zero.</p>

<a name="user-content-conv-fit"></a>
<h3><a id="user-content-conv-fit" class="anchor" aria-hidden="true" href="#conv-fit"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="#id131">Conv Fit</a></h3>
<p>ConvFit provides a simplified interface for controlling
various fitting functions (see the <a href="#id29"><span id="user-content-id30">:ref:`Fit &lt;algm-Fit&gt;`</span></a> algorithm for more
info). The functions are also available via the fit wizard.</p>
<p>Additionally, in the bottom-right of the interface there are options for doing a
sequential fit. This is where the program loops through each spectrum in the
input workspace, using the fitted values from the previous spectrum as input
values for fitting the next. This is done by means of the
<a href="#id31"><span id="user-content-id32">:ref:`ConvolutionFitSequential &lt;algm-ConvolutionFitSequential&gt;`</span></a> algorithm.</p>
<p>A sequential fit is run by clicking the Run button at the bottom of the tab, a
single fit can be done using the Fit Single Spectrum button underneath the
preview plot.</p>
<p>The fit types available in ConvFit are One <a href="#id33"><span id="user-content-id34">:ref:`Lorentzian &lt;func-Lorentzian&gt;`</span></a>, Two Lorentzian,
<a href="#id35"><span id="user-content-id36">:ref:`TeixeiraWater (SQE) &lt;func-TeixeiraWaterSQE&gt;`</span></a>, <a href="#id37"><span id="user-content-id38">:ref:`InelasticDiffSphere &lt;func-InelasticDiffSphere&gt;`</span></a>,
<a href="#id39"><span id="user-content-id40">:ref:`InelasticDiffRotDiscreteCircle &lt;func-InelasticDiffRotDiscreteCircle&gt;`</span></a>, <a href="#id41"><span id="user-content-id42">:ref:`ElasticDiffSphere &lt;func-ElasticDiffSphere&gt;`</span></a>,
<a href="#id43"><span id="user-content-id44">:ref:`ElasticDiffRotDiscreteCircle &lt;func-ElasticDiffRotDiscreteCircle&gt;`</span></a> and <a href="#id45"><span id="user-content-id46">:ref:`StretchedExpFT &lt;func-StretchedExpFT&gt;`</span></a>.</p>

<h2><a id="user-content-convfit-example-workflow" class="anchor" aria-hidden="true" href="#convfit-example-workflow"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="#id133">ConvFit Example Workflow</a></h2>
<p>The Conv Fit tab allows <code>_red</code> and <code>_sqw</code> for its sample file, and allows <code>_red</code>, <code>_sqw</code> and
<code>_res</code> for the resolution file. The sample file used in this workflow can be produced using the run
number 26176 on the <a href="#id48"><span id="user-content-id49">:doc:`Indirect Data Reduction &lt;Indirect Data Reduction&gt;`</span></a> interface in the ISIS
Energy Transfer tab. The resolution file is created in the ISIS Calibration tab using the run number
26173. The instrument used to produce these files is IRIS, the analyser is graphite
and the reflection is 002.</p>
<ol>
<li>Click <strong>Browse</strong> for the sample and select the file <code>iris26176_graphite002_red</code>. Then click <strong>Browse</strong>
for the resolution and select the file <code>iris26173_graphite002_res</code>.</li>
<li>Choose the <strong>Fit Type</strong> to be One Lorentzian. Tick the <strong>Delta Function</strong> checkbox. Set the background
to be a <strong>Flat Background</strong>.</li>
<li>Expand the variables called <strong>f0-Lorentzian</strong> and <strong>f1-DeltaFunction</strong>. To tie the delta functions Centre
to the PeakCentre of the Lorentzian, right click on the Centre parameter and go to Tie-&gt;Custom Tie and then
enter f0.PeakCentre.</li>
<li>Tick <strong>Plot Guess</strong> to get a prediction of what your fit will look like.</li>
<li>Click <strong>Run</strong> and wait for the interface to finish processing. This should generate a
_Parameters table workspace and two group workspaces with end suffixes _Results and
_Workspaces. The mini-plots should also update, with the upper plot displaying the
calculated fit and the lower mini-plot displaying the difference between the input data and the
fit.</li>
<li>Choose a default save directory and then click <strong>Save Result</strong> to save the _result workspaces
found inside of the group workspace ending with _Results. The saved workspace will be used in
the <a href="#id50"><span id="user-content-id51">:ref:`fqfit-example-workflow`</span></a>.</li>
</ol>

<p>One of the models used to interpret diffusion is that of jump diffusion in which
it is assumed that an atom remains at a given site for a time <tt>\tau</tt>; and
then moves rapidly, that is, in a time negligible compared to <tt>\tau</tt>.</p>
<p>This interface can be used for a jump diffusion fit as well as fitting across
EISF. This is done by means of the
<a href="#id92"><span id="user-content-id93">:ref:`QENSFitSequential &lt;algm-QENSFitSequential&gt;`</span></a> algorithm.</p>
<p>The fit types available in F(Q)Fit are <a href="#id94"><span id="user-content-id95">:ref:`ChudleyElliot &lt;func-ChudleyElliot&gt;`</span></a>, <a href="#id96"><span id="user-content-id97">:ref:`HallRoss &lt;func-Hall-Ross&gt;`</span></a>,
<a href="#id98"><span id="user-content-id99">:ref:`FickDiffusion &lt;func-FickDiffusion&gt;`</span></a>, <a href="#id100"><span id="user-content-id101">:ref:`TeixeiraWater &lt;func-TeixeiraWater&gt;`</span></a>, <a href="#id102"><span id="user-content-id103">:ref:`EISFDiffCylinder &lt;func-EISFDiffCylinder&gt;`</span></a>,
<a href="#id104"><span id="user-content-id105">:ref:`EISFDiffSphere &lt;func-EISFDiffSphere&gt;`</span></a> and <a href="#id106"><span id="user-content-id107">:ref:`EISFDiffSphereAlkyl &lt;func-EISFDiffSphereAlkyl&gt;`</span></a>.</p>

<a name="user-content-bayesian-fabada-minimizer"></a>
<h3><a id="user-content-bayesian-fabada-minimizer" class="anchor" aria-hidden="true" href="#bayesian-fabada-minimizer"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="#id138">Bayesian (FABADA minimizer)</a></h3>
<p>There is the option to perform Bayesian data analysis on the I(Q, t) Fit ConvFit
tabs on this interface by using the <a href="#id110"><span id="user-content-id111">:ref:`FABADA`</span></a> fitting minimizer, however in
order to to use this you will need to use better starting parameters than the
defaults provided by the interface.</p>
<p>You may also experience issues where the starting parameters may give a reliable
fit on one spectra but not others, in this case the best option is to reduce
the number of spectra that are fitted in one operation.</p>
<p>In both I(Q, t) Fit and ConvFit the following options are available when fitting
using FABADA:</p>
<dl>
<dt>Output Chain</dt>
<dd>Select to enable output of the FABADA chain when using FABADA as the fitting
minimizer.</dd>
<dt>Chain Length</dt>
<dd>Number of further steps carried out by fitting algorithm once the parameters have
converged (see <em>ChainLength</em> is <a href="#id112"><span id="user-content-id113">:ref:`FABADA`</span></a> documentation)</dd>
<dt>Convergence Criteria</dt>
<dd>The minimum variation in the cost function before the parameters are
considered to have converged (see <em>ConvergenceCriteria</em> in <a href="#id114"><span id="user-content-id115">:ref:`FABADA`</span></a>
documentation)</dd>
<dt>Acceptance Rate</dt>
<dd>The desired percentage acceptance of new parameters (see <em>JumpAcceptanceRate</em>
in <a href="#id116"><span id="user-content-id117">:ref:`FABADA`</span></a> documentation)</dd>
</dl>
<p>The FABADA minimizer can output a PDF group workspace when the PDF option is ticked. If this happens,
then it is possible to plot this PDF data using the output options at the bottom of the tabs.</p>
